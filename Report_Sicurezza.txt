1- Passaggio di dati sensibili in chiaro (ho fatto un inserimento nel DB per provare e con ad esempio questo link nella barra di ricerca <proprio_link_branch>/api/account e poi f12-->Rete-->anteprima, si possono vedere tutti i dati come username, email, password e id della persona. Questa cosa vale per tutte le rotte)
2- Mancanza Autenticazione API (ho guardato router.php e praticamente non c'è il controllo se uno è loggato. Se apro Postman e mando una richiesta DELETE a /api/persona posso cancellare chiunque anche se non ho fatto il login).
3- Auto approvazione iscrizione (in iscrizione.php ho provato a mandare nel JSON il campo "approvazione_capo": true. Il sistema lo salva senza problemi anche se non sono un capo. In pratica un utente può bypassare i permessi semplicemente scrivendo una riga nel JSON).
4- Modifica dati altrui (nelle rotte come /api/pagamento o /api/account il sistema chiede solo l'ID o l'username. Se metto quelli di un'altra persona nel JSON, posso cancellare i suoi pagamenti o cambiargli la password perché non controlla se sono veramente io).
5- Messaggi errore Troppo Dettagliati (in index.php ho visto che ci sono le righe display_errors e error_reporting. Se provo a mandare un dato sbagliato apposta, in f12-->Rete-->anteprima escono gli errori SQL che ti fanno vedere come si chiamano le tabelle e le cartelle del server).
6- Esposizione totale delle tabelle (il database restituisce sempre ogni singola colonna delle tabelle. Se per caso viene aggiunta una colonna che non deve essere visibile in questo modo viene resa visibile, quindi non bisogna usare SELECT * ma SELECT con i nomi delle colonne da usare).
7- Mancanza limite dati (Se il DB diventa grande, una semplice chiamata a /api/persona potrebbe scaricare migliaia di dati insieme, rallentando o bloccando il server).
